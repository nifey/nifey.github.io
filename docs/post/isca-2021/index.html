academia conference

ISCA is one of the premier conferences in the Computer Architecture domain.

The pandemic forced many conferences to go online, but it also made these conferences more accessible for people around the globe.

The first commercial microprocessor Intel 4004 (released in 1971) turns 50 this year. There were panel discussions on how the microprocessor industry has evolved over the 50 years and what lies ahead.

I got a chance to attend ISCA. It had a lot of interesting keynotes and panels. In this post, I’ll share the gist of some of the keynotes and panels.

Keynotes

From Transistors to Enclaves: How Architects have helped Secure Digital Assets, Healthcare records, and so much more

Hillery Hunter, CTO of IBM cloud, gave the opening keynote presentation on how IBM has integrated mechanisms for confidential computing in its newer processors. This talk is better named as “How to beat the hype cycle”

Hype Cycle

Hype Cycle is a theory of how the expectations of a new technology’s success, changes with time. It can be used to assess the maturity of a technology. It is not a rule, but an observation of the common pattern of change in expectation.
Hype Curve (image by Jeremykemp at English Wikipedia, CC BY-SA 3.0)

The hype cycle shows 5 stages through which any new technology goes through. The 5 stages are:

  1. Technology trigger
    • Some initial experiments with the technology looks promising.
    • Commercial viability of the technology is not yet known.
  2. Peak of inflated expectations
    • The hype created by the media, companies and organizations reaches its peaks
  3. Trough of disillusionment
    • Interest and investments fade away gradually
  4. Slope of enlightenment
    • In this phase, the commercially viable products get released
  5. Plateau of productivity
    • The technology’s use cases are clearly defined
    • Soon the technology is taken for granted

Confidential Computing

As cloud computing becomes more prevalent, more of our compute runs on remote machines. Data needs to be moved to untrusted machines to process. Data at rest (for instance, in a harddisk) can be encrypted. But still the data has to be decrypted shortly before it is used.

Confidential computing is about how to protect data in use when performing computation on untrusted machines.

The threat model of Trusted Execution Environments assume a privileged attacker. It should also provide attestation mechanisms to verify that the execution was indeed carried out in a TEE.

  • Confidential Computing
    • Protection of data in use by performing computing in Trusted Exeuction Environment (TEE)
    • Confidential Computing Consortium founded by IBM
  • Applications of confidential computing
    • Financial services and digital assets (crypto currency, smart contracts)
    • Healthcare records
    • Consumer data

How to beat the Hype cycle

  • Iterative (stubborn) refinement

    • Iterative improvement over existing work is not very interesting for research paper publication
    • When we set audacious goal (like landing on the moon), we know a rough trajectory to achieve that goal. But that trajectory is just a hypothesis. So it is important to refine iteratively.
    • IBM Secure Processor
      1. Plausibility Phase
      2. Prototyping Phase
      3. Realizability Phase

  • Collaboration to accomplish holistic engineering

    • Protecting sensitive data requires a holistic approach spanning compute, containers, database, etc
    • IBM Hyper protect services
      1. Data in transit (user interface)
      2. Data in use (during computing)
      3. Data at rest (in filesystems)
    • Hardware Security Module
      • Requirements
        1. Need to verify that it is running in a valid HSM and not a fake hardware
        2. Need to be secure even if adversaries carry out destructive analysis (like ColdBoot)
      • Tamper proof Crypto Co processor ASIC can help

  • Clear minded focus on use cases

    • Keeping the use case in mind can help us make better engineering trade offs
    • The pandemic has made a lot of applications move to the cloud. It is important to protect data in a shared environment like a cloud
    • Companies take 6-7 months to recover from a data breach. So they cannot afford to have a breach
    • Assurance
      • Operational assurance Cloud providers assure us that they wont access our data
      • Technical assurance Cloud providers cannot access our data (because of use of TEE)
